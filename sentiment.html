Sentiment Analysis
IMDB Movie Reviews

This project aims to to classify IMDB movie reviews into Positive and Negative.
Predicting whether a review written by a person is good or bad will be predicted using Natural Language Processing.
Reviews will be classified using Machine Learning Algorithms.

Overview

This case study basically deals with 12500 IMDB movie reviews recorded over time by different people. The sentiments recorded are either positive or negative. We have been assigned a task by the film company to predict positive and negative reviews for their upcoming film. The film company wants to count the number of Positive comments and Negative comments by different people for their analysis of the movie. So the company has assigned us this task of creating some algorithm to predict user sentiments.

Project Motivation

  • To predict user sentiments based on the past reviews recorded.
  • The solution to this problem is to create a Machine Learning model that will take the user sentiment and predict if a user review is positive or negative.
  • Hence, my motivation behind this project was to help the company by classifying user sentiments into positive or negative.

Goal of this project

  • To classify unseen positive and negative user reviews.
  • Apply NLP techniques to process the text information.
  • Apply classification algorithms to classify user reviews.

Datasets

I obtained our data set from the following link: Data Link

  • The dataset contains a resource of 25000 user sentiments for different movies.
    • The dataset is divided equally into positive and negative sentiments. There are 12500 positive reviews and 12500 negative reviews.

Steps Taken

1. Reading the data

  • Reading positive and negative reviews from files

  • Reading the data w.r.t positive and negative sentiments.

  • Creating word cloud for positive sentiment.
  • Creating word cloud for negative sentiment.

2. Data Preprocessing

  • Combining positive and negative reviews into train and test sets.

  • Removing HTML tags using BeautifulSoup.
  • Removing non-letter characters using RegEx.
  • Strip and lower case the words using strip and lower function.
  • Stem the words using NLTK.
  • Eliminating STOP words using NLTK.

3. Data Transformation

  • Normalizing the data.

4. Model Building

  • 1. Gaussian Naive Bayes Classifier

    • Gaussian Naive Bayes Classifier works better with textual data. Hence, I will be using this classification technique.


  • 2. Gradient Boosting Classifier Model

    • Testing the data for different machine learning algorithms.



3. Testing on unseen user review


5. Conclusions

  • We used Gaussian NB and Gradient Boost to classify positive and negative reviews.
  • It can be observed that both the models have almost similar accuracies.
  • Hence, we can use any of the two models to classify positive and negative user reviews.
  • Both the classifiers work correctly on the unseen user review.
  • Therefore, for this dataset we can say that we have successfully built a classification model to predict user sentiments.
